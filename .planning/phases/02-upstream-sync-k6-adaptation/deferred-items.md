# Phase 02 Deferred Items

Items discovered during Phase 02 execution that are out of scope for the
current plan but worth resolving before public-facing milestones.

## 1. `.gitignore` rules for synced + generated artifacts

**Discovered:** Plan 02-03 execution, after the real round-trip smoke test
(`npm run sync:src -- --source ../easyPlaywright --yes && npm run convert-pages`)
populated `src/pages/` (8 upstream POMs + `.sync-meta.json`) and `lib/pages/`
(8 converted POMs + `components/`).

**Issue:** These directories are NOT in `.gitignore`. After running the smoke
sequence, `git status` shows them as untracked files, inviting accidental
commits of build/sync output that should be regenerable from the upstream
sibling repo. Plan 02-02's SUMMARY explicitly forecast removing
`lib/pages/.gitkeep` once `lib/pages/base/` exists; the wipe step in
`convert-pages.mjs` performed that removal, so the only remaining concern
is that `lib/pages/*.ts` (except `base/`) and `src/pages/*.ts` should be
ignored, while `lib/pages/base/**` stays tracked.

**Recommended fix (one short patch, ~6 lines):**

```gitignore
# Synced upstream artifacts (regenerated by `npm run sync:src`)
src/pages/*
!src/pages/.gitkeep

# Generated k6-safe POMs (regenerated by `npm run convert-pages`)
lib/pages/*
!lib/pages/.gitkeep
!lib/pages/base/
!lib/pages/base/**
```

Then re-add `.gitkeep` markers under `src/pages/` and `lib/pages/` so the
empty source/target directories remain in version control.

**Why deferred:** Out of scope for Plan 02-03's locked task list. The plan
acceptance criteria do not require `.gitignore` work, and adding it now
would create scope creep mid-plan. Best resolved as a fast follow-up
(quick GSD task) before Phase 03 starts emitting scenarios that import
from `@pages/`.
